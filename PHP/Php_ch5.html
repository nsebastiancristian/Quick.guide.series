5.1. Debugging PHP in Eclipse with XAMPP and XDebug

This is how you setup Eclipse to be able to debug inside the IDE your php scripts

  1. Remember that this tutorial will work if you have XAMPP instaled on your computer. In the XAMPP control panel window go to the "Config" button and choose the "PHP(php.ini)" :
  2. Open the php.ini config file and scroll to the bottom of the file , after the [XDebug] you must uncomment by deleting the ; in front of the lines the following two lines shown by the pointers in the picture. After you uncomment the second line that is shown the picture change from "xdebug.remote_enable = 0" to "xdebug.remote_enable =1" as it is shown in the picture:
  3. If you go to back to the phpinfo() page and see a line symilar with the one marked with the arrow in the picture it means you are ready to go for debugging :
  4. Fire up Eclipse:
  5. In the menu go to File / Switch Workspace ... , choose the "Other..." part of the menu. :
  6. Choose the workspace to be inside the "htdocs" folder of the "xampp" folder:
  7. Create a new php project inside the new workspace and then go to Window / Preferences in the menu:
  8. In the new dialog box that appears expand the PHP node inside the left list and then choose the Debug option inside the sublist. On the right side of the window choose xDebug from the drop-down list next to "PHP Debugger" and uncheck the "Break at first line" checkbox :
  9. Close the dialog box by pressing finnish and then create a small test.php file with a basic for loop that echoes something and put a break point inside the loop :
  10. Right click on the new created php file and then choose "Debug As" / "PHP Web Application" from the list:
  11. There will be a message box that will advise that the best way to debug a php script is inside a "PHP Debug View" so click on "Yes" to go to that view :
  12. You will see this view after you press "Yes". This is the PHP Debug View. Here you can do all the debugging :
